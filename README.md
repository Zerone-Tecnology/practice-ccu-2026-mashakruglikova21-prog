# Практическая работа №1: Разработка Landing Page для ИТ-мероприятия

## 🎯 Цель работы

Разработать адаптивную посадочную страницу (Landing Page) для регистрации участников на технологический Хакатон. Работа направлена на освоение семантической верстки, принципов адаптивного дизайна и работы с системой контроля версий Git.

Готовые дизайны в Figma - https://www.figma.com/community/website-templates/landing-pages?resource_type=files&editor_type=all&price=all&sort_by=all_time&creators=all
Можете использовать одну из этих макетов в своей работе.

## 🛠 Технологический стек

* **HTML5**: использование семантических тегов (`<header>`, `<section>`, `<main>`, `<footer>` и т.д.).
* **CSS3**: Flexbox или Grid Layout для построения сеток.
* **Git/GitHub**: использование GitHub Classrooms для сдачи работы.

---

## 📋 Техническое задание

### 1. Структура страницы

Сайт должен состоять из следующих обязательных блоков:

1. **Header (Шапка)**:
* Логотип мероприятия.
* Навигационное меню (ссылки-якоря на разделы: О событии, Направления, Регистрация).
* Кнопка "Принять участие".


2. **Hero Section (Главный экран)**:
* Название хакатона (например, "Digital Future 2026").
* Краткое описание (слоган).
* Дата и место проведения (или формат "Online").
* Призыв к действию (CTA-кнопка).


3. **About (О мероприятии)**:
* Текстовое описание целей и задач хакатона.


4. **Tracks (Направления)**:
Страница должна содержать три карточки с описанием направлений:
* **Программирование (Web/Mobile)**.
* **Сетевые технологии (Infrastructure/DevOps)**.
* **Искусственный интеллект (AI/ML)**.


5. **Registration (Регистрация)**:
* Форма сбора данных: Имя, Фамилия, Email, Выбор направления (dropdown), Ссылка на GitHub-профиль.


6. **Footer (Подвал)**:
* Контактная информация.
* Ссылки на социальные сети.
* Копирайт.



### 2. Технические требования

* **Адаптивность**: корректное отображение на мобильных устройствах, планшетах и десктопах (используйте `@media` запросы).
* **Валидность**: код должен проходить проверку в W3C Validator.
* **Стилизация**: использование современных шрифтов (например, Google Fonts) и цветовой палитры, соответствующей ИТ-тематике.
* **Интерактивность**: плавный скролл к секциям, изменение состояния кнопок при наведении (`:hover`).

## 🚀 Инструкция по сдаче (GitHub Classrooms)

1. **Клонируйте репозиторий**:
```bash
git clone [ссылка_на_ваш_репозиторий]

```


2. **Создайте структуру файлов**:
* `index.html` — основная разметка.
* `css/style.css` — ваши стили.
* `img/` — папка для изображений.


3. **Зафиксируйте изменения**:
* Делайте осмысленные commit-сообщения (например: `feat: add registration form`, `fix: mobile navigation`).


4. **Отправьте код в GitHub**:

```bash
    git add .
    git commit -m "Final submission"
    git push origin main
    ```
